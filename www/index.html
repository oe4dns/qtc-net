QTC-Net

A Short Telegram System for Amateur Radio

QTC Net is a decentralized telegram and microblog system for amateur radio. A user or ham system can check in a telegram from any sender to any receiver, as well as access those telegrams and mark them as delivered when they are delivered.

The network itself basicly consists of signed messages that are spreaded between the network nodes. A message may contain a telegram as well as other information like the delivery notification (called qsp). The message format is designed to have a small sized footprint. It is also unencrypted therefore it is legal to transmit a message itself over a radio link.

There are gateways to DXCluster, APRS, RSS-Feed and WebPages available and many more links like DMR, PSKMail, PSK31, are planned.

Web Access URLs

https://www.qtc-net.org/ runs a small webapp which is the easies way to get access.

If you have your own application, right now there is one linux commandline thingie in the Source code, or if you want to run your own message server you can access the http interface via http://www.qtc-net.org/qtc-if.cgi in your application, or message downloader.

You may look for new messages via RSS Feed at http://www.qtc-net.org/qtc-rss.cgi?call=oe1src&call=oe1gis an optional type parameter can help you to specify what messages you are looking for

Development Status:

An alpha version as preview is released. This means the code is still changing a lot.
  • qtc::publish is a library and all you need to include to publish qtc messages
  • qtc::query is a library and all you need to include if you want to query for a set of qtc messages
  • qtc::processor is responsible for sorting all the messages into the filesytem structure
  • qtc::WebApp is an example web frontend
  • qtc::interface::http implements an incremential download interface via HTTP
  • We have an rss feed
  • qtc::aprs::is implements an APRS gateway, please start your gateway
So far if you want to see the code: qtc::msg module is already pod documented. It would be good to have that on the rest of the code as well. So if you want to contribute just make a pull request on GIT-HUB. Or write a Feedback or Bugreport. All bugs are filed via GitHubs BTS, we are also using that list as a Tasklist so we do not forget anything.
